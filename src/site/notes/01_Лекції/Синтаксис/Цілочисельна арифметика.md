## Множинне присвоєння

Python дозволяє присвоїти значення декільком змінним одночасно. Для цього використовується наступний синтаксис:

``` python
num1, num2 = 10, 20
```

> [!light-yellow] Пояснення
> Зліва, через кому, записуються змінні, яким потрібно присвоїти значення. Справа, так само через кому, записуються значення які потрібно присвоїти у тому ж порядку, у яку змінну присвоюється значення.

Якщо потрібно для декількох змінних присвоїти один і той самий об'єкт це записується ось так:

``` python
num1 = num2 = num3 = 100
```

> [!attention] Зверніть увагу
> В даному випадку всі змінні посилаються на один і той самий об'єкт. 

> [!danger] Важливе уточнення 
> Синтаксис Python не дозволяє вказувати тип даних (`int`, `str`) безпосередньо під час множинного присвоєння. Тип змінних визначається автоматично — він залежить від значень або функцій, що стоять праворуч від знака дорівнює `=`

## Введення декількох значень в один рядок

Стандартна функція `input()` створює об'єкт в пам'яті типу `str` незалежно від того, що саме було подано на вхід. Саме тому запис `10 20 30` буде сприйматись як єдине ціле, а не три числа записаних через пробіл.

Для того щоб розділити ці значення використовується метод `.split()`. В нього передається роздільник(умова за якою відділяє об'єкти між собою, в прикладі вище це пробіл) і утворюється об'єкт типу `list`(вони будуть розглядатись пізніше).

Але навіть після застосування методу `.split()` це все ще буде один об'єкт-контейнер з текстом в середині і ми не можемо одразу конвертувати їх в інший тип даних стандартними функціями перетворення типів. Для того щоб конвертувати об'єкти в потрібний тип даних використовується функція `map()`.

**Шаблон введення двох чисел розділених через пробіл**

``` python
num1, num2 = map(int, input().split())
```

> [!faq] Уточнення
> В методі `.split()` можна передавати не лише пробіл, будь що може бути роздільником. Як розділові знаки(`;` `.` `:` `,` `!`) так і букви, слова або інші спец. символи. Головне вказати їх в середині `()` обов'язково в лапках `""` . Якщо дужки пусті то розділення відбувається автоматично за пробілами, незалежно від того скільки їх буде поставлено між елементами

>[!light-green] Розбір коду
>- зліва, через кому, вказуються змінні які потрібно присвоїти
>- після оператора присвоєння викликається функція `map()`
>- в середині функції передається два аргументи: тип даних в який потрібно конвертувати та самі об'єкти через функцію `input()` з застосованим методом `.split()`

> [!danger] Увага
> Кількість змінних обов'язково має відповідати кількості об'єктів які будуть передані в `input()` інакше програма аварійно завершиться

## Зняття обмежень за допомогою модуля sys

Python підтримує `довгу арифметику`, але за замовчуванням встановлено обмеження на перетворення (конвертацію) чисел, довших за 4300 цифр, у рядок (для виведення) або з рядка. Для багатьох завдань цього цілком достатньо, але у випадку коли потрібно працювати з гігантськими степенями або факторіалами це обмеження може призвести до обвалу програми навіть при правильній  реалізації алгоритму. 

Для зняття цих обмежень використовується модуль `sys`

``` python
import sys

sys.set_int_max_str_digits(0)
```

> [!light-yellow] Пояснення
> Достатньо прописати цих два рядки на початку програми щоб зняти обмеження на довжину рядкового представлення числа.

## Цілочисельне ділення

Раніше вже було розглянуто, що оператор ділення `/` в результаті створює об'єкт типу `float`
Однак існують ситуації коли потрібно отримати саме цілу частину від ділення. Один з варіантів який можна використати це функція `int()` щоб перетворити отриманий результат на ціле число шляхом відкидання дробової частини.

Але такий підхід потребує додаткових ресурсів на виконання програми і написання додаткового коду, а ситуацій де потрібно знаходити саме цілу частину від ділення настільки багато, що в мовах програмування для цього існує окремий оператор.

В python він має вигляд `//` і в результаті обчислень повертає цілу частину від ділення шляхом заокруглення до меншого цілого з типом `int`.

``` python
num: int = 108 // 5
print(num) # виведе 21
```

Якщо ж ми розглядаємо ділення від'ємного числа на додатне, то слід враховувати, що при цілочисельному діленні python заокруглює до меншого цілого числа. Саме тому `-10 // 3` видасть `-4`:
- -10 / 3 = -3.33...
- python аналізує результат ділення і бачить, що число від'ємне
- Тепер він заокруглює в меншу сторону(до $-\infty$) 
- Тому в результаті -3.33... перетворюється на -4

> [!danger] Увага
> В даному випадку цілочисельне ділення `//` видасть інший результат ніж `int(-10 / 3)` так як функція `int()` не виконує округлень, а просто відкидає дробову частину.

## Ділення з остачею

Ділення одного натурального числа на інше не завжди можна виконати націло, тому розглядається ділення з остачею. 

> [!light-green] Ділення з остачею
> $$A : B = Q (остача R)$$

Де:
- $A$ — **Ділене** (число, яке ділимо).
- $B$ — **Дільник** (число, на яке ділимо).
- $Q$ — **Частка** (результат цілочисельного ділення).
- $R$ — **Остача** (те, що не поділилося).

Теорема про ділення з остачею звучить так:

> [!teorema] Теорема
> Для будь-яких натуральних чисел `A` і `B` існує єдина пара невід’ємних цілих чисел `Q` і `R` , таких що
> $$A = B \cdot Q + R$$

Звідси випливає формула остачі:

> [!formula] Формула остачі
> $$R = A - B \cdot Q$$

> [!danger] Увага
> Остача завжди набуває невід'ємних значень $0 \le R < B$

Саме за цією формулою працює оператор ділення з остачею в python `%`. Адаптуємо запис формули до того як виконується обчислення вже в python:

> [!light-purple] Формула остачі в python
> r = a - (a  //  b)  *  b
> 

Використовується коли потрібно знайти залишок від ділення.

> [!faq] Уточнення
> В python остача має знак дільника

Приклад з додатними числами

``` python
num1: int = 108 % 5
print(num1)
```

Пояснення
- оператор ділення з остачею `%` бере числа по краях і підставляє їх у формулу остачі
- Отримаємо: num1 = 108 - (108 // 5) * 5
- 108 // 5 дасть нам 21
- num1 = 108 - 21 * 5
- num1 = 108 - 105
- num1 = 3

Приклад з від'ємним діленим

``` python
num2: int = -5 % 108
```

Пояснення
- Підставляємо числа у формулу: -5 - (-5 // 108) * 108
- -5 // 108 дасть нам -1
- -(-1) * 108 дасть нам 108
- -5 + 108 дасть нам 103

Отже, остача від ділення від'ємного числа на додатне дасть нам число додатне, що не суперечить математичному визначенню остачі. 

> [!faq] Уточнення
> Якщо ж дільник також буде від'ємним (ділення двох від'ємних чисел), результат остачі теж буде від'ємним (бо в Python знак остачі залежить від дільника). На практиці такі ситуації зустрічаються вкрай рідко і розглядаються окремо.
