## Пам'ять

Для роботи програм в реальному часі використовується *оперативна пам'ять*.

> [!term] Оперативна пам'ять
>  Пам'ять з випадковим(рандомним) доступом, скорочено RAM.
>  Високошвидкісна енергозалежна пам'ять комп'ютера, що тимчасово зберігає дані програм з якими комп'ютер працює в певний момент часу. 
>  

>[!INFO] Чим більше оперативної пам'яті тим з більшою кількістю програм комп'ютер може працювати одночасно

## Об'єкт

> [!term] Об'єкт у Python
> Це базова одиниця даних, що розміщується в оперативній пам'яті комп'ютера.

Абсолютно всі сутності, якими оперує програма це об'єкти:

- Конкретні числа (наприклад, `10`, `-5`).
- Рядки тексту (наприклад, `"Hello World"`).
- Функції та складніші структури даних.

Кожен об'єкт — це екземпляр певної сутності, що займає виділену ділянку пам'яті.
## Властивості об'єктів

Кожен об'єкт в Python обов'язково має три фундаментальні характеристики:

>[!light-yellow] Ідентифікатор (Identity)
>Унікальна числова адреса об'єкта в оперативній пам'яті. Це "паспорт" об'єкта, який не змінюється протягом його життя.

> [!light-green] Тип (Type)
> Характеристика, що визначає клас об'єкта. Тип вказує яких значень може набувати об'єкт і які операції над ним можна виконувати

> [!light-purple] Значення (Value)
> Дані, що зберігаються в об'єкті

Процес роботи з пам'яттю включає:

- **Алокація (Виділення):** При створенні нового об'єкта система знаходить вільний блок пам'яті потрібного розміру і розміщує об'єкт там.

- **Життєвий цикл:** Об'єкти існують у пам'яті лише доти, доки вони потрібні програмі (на них є активні посилання).

- **Garbage Collection (Збирання сміття):** Якщо об'єкт втрачає всі посилання (стає недосяжним для коду), інтерпретатор Python автоматично звільняє зайняту ним пам'ять для інших потреб.

## Змінна

> [!WARNING] Ключова відмінність Python 
> На відміну від багатьох мов (C++, Pascal), де змінна є "контейнером" для даних, у Python змінна є лише **іменованим посиланням (reference)** на об'єкт.

**Змінна** не містить даних всередині себе. Вона зберігає лише адресу місця в пам'яті, де лежить конкретний об'єкт. Це просто ім'я, яке ми використовуємо для доступу до даних.

``` mermaid
graph LR
    A[Ім'я змінної: x] -->|Посилання| B(Об'єкт у пам'яті: 10);
    style B fill:#f9f,stroke:#333,stroke-width:2px,color:black
    style A fill:#ccf,stroke:#333,stroke-width:2px,color:black
```

#### Динамічна типізація та переприсвоєння

Створення зв'язку між іменем та об'єктом відбувається через операцію **присвоєння** (символ `=`).

Розглянемо інструкцію `score = 100`:

1. **Створення об'єкта:** У пам'яті створюється об'єкт цілого числа зі значенням `100`.
2. **Створення імені:** У поточному просторі імен створюється ідентифікатор `score`.
3. **Зв'язування:** Ім'я `score` починає посилатися на новостворений об'єкт `100`.

``` mermaid
graph TD
    %% Крок 1
    Step1(("1️⃣ Крок 1:<br/>Створюємо 100"))
    
    %% Крок 2
    Step2["2️⃣ Крок 2:<br/>Пишемо ім'я 'score'"]
    
    %% Крок 3 (Дія)
    Action{3️⃣ Крок 3:<br/>З'єднуємо}

    %% Зв'язки
    Step2 --> Action
    Step1 --> Action
    Action --> Result["Готова змінна: score ➔ 100"]

    %% Стилі
    style Step1 fill:#FFF59D,stroke:#FBC02D
    style Step2 fill:#E3F2FD,stroke:#1E88E5
    style Result fill:#C8E6C9,stroke:#43A047,stroke-width:2px
```


Якщо ми потім напишемо `score = 200`:

4. Створюється новий об'єкт `200`.
5. Посилання `score` "перемикається" на новий об'єкт.
6. Старий об'єкт `100` (якщо на нього більше ніхто не посилається) буде знищено збиральником сміття.


``` mermaid
graph LR
    A[Змінна: score] -.->|Старе посилання розірвано| B(Об'єкт: 100);
    A -->|Нове посилання| C(Новий Об'єкт: 200);
    style B fill:#ccc,stroke:#333,stroke-width:1px,stroke-dasharray: 5 5
    style C fill:#f9f,stroke:#333,stroke-width:2px,color:black
    style A fill:#ccf,stroke:#333,stroke-width:2px,color:black
```

Таким чином змінні в python це посилання на об'єкти в оперативній пам'яті.
Однак ця особливість роботи з даними у python відрізняється від принципу роботи в інших мовах програмування і може викликати плутанину при переході на інші технології. Втім, загальний принцип того як писати код для більшості мов однаковий і потрібно лише враховувати нюанси конкретної мови. 

## Типи даних (Концепція)

Комп'ютер — це неймовірно педантична машина. Він має знати природу інформації, з якою працює, заздалегідь. Він не може просто "здогадатися", що `15` — це кількість яблук, а `"Привіт"` — це привітання.

Для комп'ютера різні дані — це як різні фізичні стани речовини (тверде, рідке, газоподібне). Щоб не виникало хаосу, всі дані у програмуванні поділяються на **Типи**.

> [!TERM] Тип даних 
> Це властивість об'єкта, яка повідомляє комп'ютеру:
> 
> 1. Що це за дані (число, текст і т.п.).
>     
> 2. Скільки місця об'єкт займає в пам'яті.
>     
> 3. Які дії з ним дозволено робити.
>     

Ми почнемо з чотирьох "китів", на яких тримається вся логіка програм.

### 1. Цілі числа (Integer)

Це все, що можна **полічити поштучно**. Ці об'єкти неподільні в контексті лічби. Уявіть собі цеглинки лего або людей у кімнаті. Ми не можемо сказати "півтори людини" або "3.5 цеглинки" (якщо ми їх рахуємо).

> [!EXAMPLE] Приклади з життя
> 
> - Кількість патронів у грі.
>     
> - Номер поверху в ліфті.
>     
> - Кількість підписників в Instagram.
>     
> - Рік народження.
>     

### 2. Дійсні / Дробові числа (Float)

Це все, що ми **вимірюємо**. Такі дані можуть бути неточними, мати "хвостик" після коми. Уявіть собі воду або пісок. Ви можете відміряти 1 літр, а можете 1.5 літра або 1.25 літра. Це неперервні величини.

> [!EXAMPLE] Приклади з життя
> 
> - Вага персонажа (85.5 кг).
>     
> - Швидкість автомобіля (60.2 км/год).
>     
> - Температура повітря (-2.5 градуси).
>     
> - Ціна товару (19.99 грн).
>     

### 3. Рядкові дані / Текст (String)

Це будь-яка інформація, записана символами. Це те, що ми читаємо та пишемо. Для комп'ютера текст — це просто набір символів, нанизаних на ниточку, як намисто. Навіть якщо ви напишете цифри як текст (наприклад, номер телефону), для комп'ютера це будуть просто малюнки, які не можна додавати чи множити математично.

> [!EXAMPLE] Приклади з життя
> 
> - Нікнейм у грі ("SuperSteve").
>     
> - Повідомлення в чаті ("GG WP").
>     
> - Пароль від Wi-Fi.
>     
> - Номер будинку ("22-А").
>     

### 4. Логічний тип (Boolean)

Це найпростіший тип, який нагадує звичайний **вимикач світла**. У нього є лише два положення. Жодних "можливо", "скоріше за все" або "майже". Тільки конкретика.

> [!EXAMPLE] Приклади з життя
> 
> - Світло: Увімкнено / Вимкнено.
>     
> - Відповідь на тесті: Правильно / Неправильно.
>     
> - Доступ до сайту: Дозволено / Заборонено.
>     
> - Персонаж живий? Так / Ні.
>

### Чому це важливо? 

Уяви, що дані — це продукти, а змінні — це контейнери на кухні.

- **Цілі числа** — це яблука. Їх кладуть у кошик.

- **Дробові числа** — це вода. Для неї потрібна пляшка.

- **Текст** — це етикетка з рецептом. Її клеять на холодильник.

Якщо ти спробуєш налити воду (Float) у паперовий пакет для записок (String), або покласти яблуко (Int) у пляшку з вузьким горлом — нічого не вийде. Саме тому ми спочатку розбираємося з типами, а вже потім створюємо змінні

Важливо розуміти, що зовнішня схожість даних для людини не означає однаковість для комп'ютера. 

> [!INFO] Інтерпретація даних 
> Тип даних діє як "лінза", крізь яку комп'ютер дивиться на об'єкт. Одна й та сама сутність сприймається по-різному залежно від типу.

> [!WARNING] Важливо 
> Тип даних незмінно пов'язаний з об'єктом у момент його створення. Спроба виконати математичну операцію над текстом (наприклад, відняти одне слово від іншого) призведе до помилки, оскільки це суперечить природі типу `String`.
